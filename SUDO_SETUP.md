# 🔑 Запуск скриптов без прямого подключения как root

## Проблема (решена)

Некоторые системы не позволяют прямое SSH подключение как root (что является корректной практикой безопасности).

## ✅ Решение

Все скрипты теперь работают от обычного пользователя с доступом к `sudo`.

## 🚀 Как использовать

### 1. Убедитесь что ваш пользователь в группе sudo
```bash
# Проверка - если команда выполнится, у вас есть доступ к sudo
sudo -n true
# Если запросит пароль - это нормально
```

### 2. Запустите скрипты с sudo
```bash
# От вашего пользователя (НЕ от root!)
sudo ./scripts/deploy.sh all .deployment.conf
```

**Примечание:** Скрипт запросит пароль sudo если требуется.

### 3. Опционально: Отключите требование пароля для sudo

Если вы хотите чтобы скрипты работали без запроса пароля, отредактируйте `sudoers`:

```bash
# ОПАСНО: Используйте только в доверенных сетях/окружениях!
sudo visudo

# Найдите строку:
# %sudo ALL=(ALL:ALL) ALL

# И замените на:
%sudo ALL=(ALL:ALL) NOPASSWD: ALL
```

**Или более безопасный вариант** - разрешить только спецефические команды для развёртывания:

```bash
sudo visudo

# Добавьте в конец файла:
ceph ALL=(ALL) NOPASSWD: /usr/bin/sudo /bin/bash /tmp/ceph-deploy/scripts/*
```

## 📋 Что изменилось в скриптах

Все скрипты изменены с:
```bash
# БЫЛО (требует прямого root)
if [[ $EUID -ne 0 ]]; then
   echo "Этот скрипт должен быть запущен с правами root"
   exit 1
fi
```

На:
```bash
# СТАЛО (работает с sudo)
if ! sudo -n true 2>/dev/null; then
   echo "Требуется доступ к sudo"
   sudo true || { echo "Нет доступа к sudo"; exit 1; }
fi
```

## ✨ Преимущества этого подхода

✅ **Безопаснее** - нет необходимости в SSH доступе как root  
✅ **Логирование** - все команды sudo логируются в syslog  
✅ **Гибче** - можно ограничить доступ к спецефическим командам  
✅ **Стандартнее** - это рекомендуемая практика в современных системах  

## 🔍 Если что-то не работает

### Ошибка: "Нет доступа к sudo"
```bash
# Проверьте что пользователь в группе sudo
groups $USER

# Если его там нет, добавьте его (нужны права root):
sudo usermod -aG sudo $USER

# Перезагрузитесь или переподключитесь через SSH
```

### Ошибка: "sudo: пароль для пользователя:"
Это нормально! Просто введите ваш пароль.

### Ошибка: "Permission denied" при выполнении команды
Это может означать что ваш пользователь не может выполнить конкретную команду через sudo. Решение - отредактируйте sudoers как описано выше.
