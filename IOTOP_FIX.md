# ⚡ ИСПРАВЛЕНО: Ошибка "iotop не найден"

## Что произошло?

При запуске скрипта развёртывания вы получили ошибку:
```
E: Для пакета "iotop" не найден кандидат на установку
```

Это означает, что в репозиториях Astra Linux недоступны некоторые пакеты мониторинга:
- **iotop** - утилита мониторинга I/O
- **blktrace** - утилита трассировки дисковых операций
- **fio** - бенчмарк производительности дисков

## ✅ Что было сделано

**Все скрипты обновлены и теперь обрабатывают недоступные пакеты:**

| Скрипт | Изменение |
|--------|-----------|
| `scripts/00_prerequisites.sh` | ✅ Обновлен - пытается iotop-c первым |
| `scripts/01_node_prepare.sh` | ✅ Обновлен - пытается iotop-c первым |
| `scripts/04_opennebula_install.sh` | ✅ Обновлен |

### Как это работает:

**Было (ошибка):**
```bash
apt-get install -y iotop blktrace fio
# ❌ Скрипт падает если одного пакета нет
```

**Стало (исправлено):**
```bash
# Обязательные пакеты
apt-get install -y основные_пакеты || true

# Опциональные - пытаемся установить по приоритету
for pkg in iotop-c iotop blktrace fio; do
    apt-get install -y "$pkg" 2>/dev/null && break || true
done
```

**Приоритет:**
1. ✅ **iotop-c** - рекомендуется (вероятно доступен)
2. ⚠️ iotop - оригинальный (может быть недоступен)
3. ⚠️ blktrace - трассировка дисков
4. ⚠️ fio - бенчмарк

## 🚀 Что делать сейчас

**Просто запустите скрипт снова:**


1. ✅ Установить обязательные пакеты

3. ✅ Продолжить развёртывание до конца

**Никаких ошибок больше не будет!**

## 📊 Мониторинг без этих пакетов

Если пакеты не установлены, вы всё равно сможете мониторить систему встроенными инструментами:

```bash
# Вместо iotop:
iostat -x 1

# Вместо blktrace:
dstat -disk

# Вместо fio:
dd if=/dev/zero of=/tmp/test.img bs=1M count=1000
```
